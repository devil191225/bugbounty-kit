To capture Flag 3 in the Hacker101 Postbook challenge, you need to use your browser's Inspect Element tool to manipulate a hidden form field during post creation. This teaches you how to test for Insecure Direct Object Reference (IDOR) and client-side validation flaws.Steps to Capture Flag 3Navigate to the Post Creation: Log into the Postbook application and click the button to create a new post.Inspect the Form: Right-click anywhere near the post input box or form and select Inspect (or Inspect Element) to open the developer tools.Find the Hidden Field: Look through the HTML code for an input field that sets the user ID. It usually looks like this: <input type="hidden" name="user_id" value="2">.Change the Value: Double-click on the value attribute and change it from your current user ID to a different number, such as 1 (which is typically the administrator ID).Submit the Post: Type some test content into your post and click Create Post.Reveal the Flag: By successfully tricking the server into thinking the post was created by the administrator, the flag will be revealed.
